# APEX — Конструктор КП

Веб-инструмент для быстрой сборки коммерческих предложений агентства APEX: форма слева, живой предпросмотр документа справа, экспорт в PDF, история всех сформированных КП со статусами.

Чистый HTML/CSS/JS без сборки и бэкенда — открывается прямо в браузере или как статический сайт (например, GitHub Pages).

## Возможности

- Конструктор КП: клиент, ниша, боль клиента, произвольная смета (разово / в месяц), три кейса агентства с возможностью отключить любой.
- Живой предпросмотр документа в фирменном стиле APEX, редактируемые прямо в превью блоки («Цель сотрудничества», «Условия работы»).
- Экспорт в PDF (html2canvas + jsPDF), с автоматической нарезкой на страницы A4.
- История КП: все сохранённые предложения со статусами (Черновик / Отправлено / На согласовании / Принято / Отклонено), сводный дашборд по количеству и суммам.
- Действия из истории: открыть на редактирование, дублировать (создаёт новое КП с новым номером), удалить.

## Хранение данных

Все данные (список КП, статусы, сквозной номер) хранятся в `localStorage` браузера. Сервера и базы данных нет — это значит:

- История видна только в том браузере и на том устройстве, где создавались КП.
- Открытие ссылки в другом браузере / режиме инкогнито / на другом устройстве даёт **пустую** историю.
- Очистка данных браузера (или localStorage конкретного сайта) сотрёт историю КП безвозвратно.

Если в будущем понадобится общая история для всей команды — потребуется подключить облачное хранилище (например, Firebase или Supabase) вместо localStorage.

## Локальный запуск

Открыть `index.html` напрямую в браузере, либо поднять любой статический сервер из корня проекта:

```bash
python3 -m http.server 8080
# затем открыть http://localhost:8080
```

## Деплой на GitHub Pages

1. Создать репозиторий на GitHub и запушить в него содержимое этой папки (ветка `main`).
2. В репозитории: **Settings → Pages → Build and deployment → Source: Deploy from a branch**, branch `main`, папка `/ (root)`.
3. Через минуту сайт будет доступен по адресу вида `https://<username>.github.io/<repo>/`.

## Структура проекта

```
index.html          разметка и подключение стилей/скриптов
css/style.css        все стили
js/data.js            статические данные: кейсы, контакты, статусы КП
js/util.js            форматирование, экранирование, toast-уведомления
js/storage.js         localStorage: CRUD по КП, счётчик номеров
js/constructor.js     форма, живой рендер документа, сохранение, экспорт PDF
js/history.js         вкладка "История КП": дашборд и таблица
js/app.js             переключение вкладок, инициализация
```
